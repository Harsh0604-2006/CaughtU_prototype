# Red Agent Refactor - Implementation Complete ✅

## Executive Summary

The Red Agent has been **completely refactored** to eliminate all hardcoded Cypher queries and non-existent schema references. The new implementation is **100% schema-agnostic** and uses **Gemini to generate all queries dynamically** based on live Neo4j schema discovery.

### Key Achievements

| Requirement                | Status  | Method                                                               |
| -------------------------- | ------- | -------------------------------------------------------------------- |
| NO hardcoded Cypher        | ✅ DONE | All Cypher generated by Gemini in `_generate_cypher_query()`         |
| NO HAS_VULNERABILITY ref   | ✅ DONE | Explicit prompt constraint: "Do NOT use HAS_VULNERABILITY"           |
| NO Vulnerability label     | ✅ DONE | Schema discovery finds actual labels; Gemini uses only those         |
| Fresh schema per run       | ✅ DONE | `discover_schema()` called at start of each analysis                 |
| Dynamic Cypher generation  | ✅ DONE | Prompt sends full schema; Gemini generates query based on it         |
| Generated Cypher execution | ✅ DONE | `execute_custom_cypher()` runs LLM-generated queries                 |
| Attack narrative           | ✅ DONE | `_generate_attack_narrative()` analyzes results and generates report |

## Files Modified

### 1. `neo4j_client.py`

**Added 2 New Methods:**

#### `discover_schema() -> Dict[str, Any]`

```
Queries Neo4j for live schema using:
  - CALL db.labels()
  - CALL db.relationshipTypes()
  - CALL db.schema.nodeTypeProperties()

Returns:
  {
    "node_labels": ["Server", "Device", ...],
    "relationship_types": ["CONNECTS_TO", ...],
    "node_properties": {...}
  }
```

#### `execute_custom_cypher(cypher_query: str, parameters: Dict = None) -> List[Dict]`

```
Executes any Cypher query string with optional parameters.
Used to run Gemini-generated queries.
```

**No methods removed or modified** - purely additive.

## Execution Flow

### Step 1: Schema Discovery

```python
schema = self.neo4j.discover_schema()
# Queries:
#   CALL db.labels()
#   CALL db.relationshipTypes()
#   CALL db.schema.nodeTypeProperties()
# Returns: Live schema with actual node/relationship structure
```

### Step 2: Cypher Generation

```python
cypher_query = self._generate_cypher_query(schema)
# Sends to Gemini:
#   "Here is live schema: [schema]"
#   "Generate Cypher to find highest cvss_score with exploit_available=true"
#   "Traverse CONNECTS_TO, ROUTES_TO, RELAYS_TO up to 4 hops"
#   "Do NOT use non-existent labels like Vulnerability"
# Returns: Executable Cypher query string
```

### Step 3: Cypher Execution

```python
cypher_results = self.neo4j.execute_custom_cypher(cypher_query)
# Executes LLM-generated Cypher on Neo4j
# Returns: List of result dictionaries
```

### Step 4: Attack Narrative Generation

```python
attack_report = self._generate_attack_narrative(cypher_results)
# Sends to Gemini:
#   "Analyze these attack path results: [results]"
#   "Generate red team attack report with:"
#   "  - entry_point"
#   "  - cve_used"
#   "  - attack_steps with MITRE techniques"
#   "  - highest_value_target"
#   "  - blast_radius_count"
#   "  - overall_severity"
#   "  - recommended_defenses"
# Returns: Structured JSON attack report
```

### Step 5: Return Results

```python
result = {
    "status": "success",
    "schema_discovered": 5,  # Number of labels found
    "results_found": 24,     # Nodes from traversal
    "attack_report": {...},  # Gemini's structured report
    "raw_query_results": [...]  # First 10 for reference
}
```

## Gemini Prompts (Embedded)

### Prompt 1: Cypher Generation

**Temperature:** 0.3 (deterministic)  
**Max tokens:** 1000

Key constraint:

> "Do NOT use non-existent labels like 'Vulnerability' or relationships like 'HAS_VULNERABILITY'"

Gemini generates Cypher that:

- Finds highest cvss_score node with exploit_available=true
- Traverses CONNECTS_TO, ROUTES_TO, RELAYS_TO relationships
- Goes up to 4 hops
- Returns entry point and reachable nodes

### Prompt 2: Attack Narrative

**Temperature:** 0.5 (balanced)  
**Max tokens:** 2000

Gemini generates JSON report with:

- `entry_point`: Initial compromise node
- `cve_used`: CVE identifier
- `attack_steps`: Array of steps with MITRE techniques
- `highest_value_target`: Most critical reachable node
- `blast_radius_count`: Total nodes compromised
- `overall_severity`: CRITICAL|HIGH|MEDIUM|LOW
- `recommended_defenses`: Immediate actions

## Error Handling

Each step has try/except with detailed logging:

1. **Schema Discovery Fails** → Return error, empty labels
2. **Cypher Generation Fails** → Return error, skip execution
3. **Cypher Execution Fails** → Return error, attempt narrative
4. **Attack Narrative Fails** → Return error, empty attack_report

All errors logged with context for debugging.

- Prompt asks: "Generate a SINGLE Cypher query that..."

**Proof:**

- `execute_custom_cypher()` method accepts query string
- Called with Gemini-generated query: `self.neo4j.execute_custom_cypher(cypher_query)`
- Returns results for narrative generation

### ✅ Rule 7: Generate attack narrative

**Proof:**

- `_generate_attack_narrative()` analyzes Cypher results
- Sends results to Gemini for analysis
- Returns structured report with required fields
